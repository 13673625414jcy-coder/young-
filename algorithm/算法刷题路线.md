# AI 全栈求职：算法一个月冲刺题单

周期：2026-05-12 至 2026-06-11  
目标：以面试高频题为主，按知识点建立题型记忆，不追求刷完所有题，优先做到核心题能独立写出、能讲清楚。

数据来源：
- LeetCode 热题 100：https://leetcode.cn/studyplan/top-100-liked/
- CodeTop 高频榜：https://codetop.cc/home
- CodeTop API 口径：按 `frequency` 降序，优先选择出现次数大于等于 80 的题目。

## 刷题规则

每天按这个闭环来，不要只提交通过：

1. 先看题，5 分钟内判断主知识点和复杂度目标。
2. 独立写 20-35 分钟，卡住再看提示。
3. 通过后立刻写 5 行复盘：模板、边界、复杂度、错因、相似题。
4. 第 2 天、第 7 天各重写一次核心题。
5. 面试前要能口述：为什么这样做、有没有替代方案、边界怎么处理。

优先级说明：
- S：必须掌握。CodeTop 高频或 Hot100 核心，面试非常常见。
- A：建议掌握。常见变体或某类题模板题。
- B：补充。时间足够再刷，或者作为专项加强。

## 面试前必须背熟的模板

1. Map/Set：两数之和、最长连续序列、前缀和计数。
2. 滑动窗口：右指针扩张，左指针收缩，何时更新答案。
3. 双指针：排序后对撞、快慢指针、链表双指针。
4. 链表：虚拟头、反转链表、区间反转、快慢指针找中点。
5. 栈：括号匹配、单调栈、字符串解码。
6. 二分：闭区间 `[l, r]` 和 lowerBound 模板。
7. 树：前中后序、层序、后序返回值、全局答案。
8. 回溯：路径、选择列表、结束条件、撤销选择。
9. 图：网格 DFS、BFS 队列、拓扑排序。
10. DP：状态定义、转移方程、初始化、遍历顺序。

## 风险与取舍

- 不建议从第 1 天就啃大量 Hard。Hard 放在第 4 周专项处理，否则会打乱信心。
- 如果每天只有 1 小时，优先刷 S 级题，A/B 级直接跳过。
- 如果某类连续卡住，先看模板再刷同类 3 题，不要跨知识点乱跳。
- 前端 / AI 全栈面试里算法通常不是最重，但高频链表、数组、滑窗、树必须守住。
- JavaScript 写算法要注意：数组模拟队列不要频繁 `shift()`，大数据用指针；对象 key 和 Map 的区别要讲清楚。
